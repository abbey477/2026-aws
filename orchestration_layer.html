Orchestration Layer — RDR Cloud

The orchestration layer is managed by AWS Step Functions. It handles job routing, async triggering of ECS and Databricks workloads, monitoring, and state updates via DynamoDB.

Diagram

Flow Summary

StepDescription
Get configReads job configuration from DynamoDB
Job runner?Branches on runner type: S32S or S2S3
Run ECS taskLambda async-triggers ECS Fargate and returns immediately
Run DBR jobLambda async-triggers Databricks job and returns immediately
ECS FargateRuns parallel processing, writes status to DynamoDB on completion
DatabricksRuns DBR job independently (no direct DynamoDB write)
Wait & MonitorEventBridge scheduler polls Databricks API, writes result to DynamoDB
DynamoDBSingle source of truth — stores config, job status, and final state

Key Design Decisions

  • Async triggers — both ECS and Databricks are fire-and-forget; no Lambda timeout risk
  • DynamoDB as state store — single source of truth for all job statuses
  • EventBridge scheduler — drives the Wait & Monitor polling loop without Step Functions native wait
  • Databricks status via API — Wait & Monitor polls Databricks REST API and writes result to DynamoDB
  • Step Functions boundary — Get config, Job runner, Run ECS task and Run DBR job are all orchestrated within AWS Step Functions
